iFox.Resume&Go
Reference

Properties
Name Description
TransferType

Value type: Integer
Default value: TYPE_FILE

If the value of this property is TYPE_FILE, download proceeds right to the file indicated by the FileName property.

If the value of this property is TYPE_MEMORY, download proceeds in memory and it can be retrieved with the method GetPartialDownload.
FileName

Value type: String

If the value of TransferType property is TYPE_FILE, this property must contain the local file name where the remote file will be downloaded.

If the value of TransferType property is TYPE_MEMORY, this property is ignored.
FileSize

Value type: Long

After a succesful call to the StartHTTPTransfer method this property will hold the byte size of the file to download.
DownloadSize

Value type: Long

After a succesful call to the StartHTTPTransfer method this property will hold the byte count to download.

In case of a partial transfer, FileSize property will indicate the total file size and DownloadSize the remaining bytes to download.
DownloadComplete

Value type: Logical

Indicates when transfer is complete.
DownloadedBytes

Value type: Long

Indicates the quantity of bytes transferred until now. This property value will vary between zero and the value indicated by the DownloadSize property.
PreviousDownload

Value type: Long

Indicates the amount of bytes transferred in a previous download. The sum of this property and the value of DownloadedBytes property will indicate the total size downloaded in all sessions.
ErrorNumber Value type: Long

In case of an error, this property will hold the related error number. Possible values are:

1. Unable to initialize iFox.Sockets.
2. Unable to determine the server IP address.
3. Unable to establish communication with the given server.
4. No response from server.
5. Server doesn't support the HTTP 1.1 protocol.
6. Server doesn't support partial downloads.
7. File is already completely downloaded.
8. Local file has not being indicated in FileName property.
9. Unable to open the file indicated by FileName property.
10. Response from server is invalid.

101. Communication error (only for DownloadNextPart).
102. Write error (only for DownloadNextPart).


Methods
Name Description
StartHTTPTransfer

Parameters: cServer as String, cURL as String, nStart as Long (optional), nPort as Integer (optional), nTimeOut as Integer (optional), lDynamic as Boolean (optional)

Returns: Logical

Returns the content of the URL indicated by cURL parameter. The server part of the URL must be indicated in the cServer parameter.

The nStart parameter indicates the position where the download start, if not indicated the file is fully downloaded.

The nPort parameter indicates the port number to use, if not indicated the default HTTP port (80) will be used.

The nTimeOut parameter indicates the time (in seconds) to wait for the server response, if not indicated the component will wait for 15 seconds.

Sometimes iFox.Resume&Go cannot download a file generated by a dynamic page (by ASP, PHP, etc.) because of the different HTTP headers received. You must set the lDynamic parameter to true if you're having trouble with dynamically generated downloads.

DownloadNextPart

Parameters: None

Returns: String

Continues a transfer initiated by StartHTTPTransfer.

Return value indicates if the operation has being succesful.
GetZeroCount

Parameters: None

Returns: Integer

During tests on memory handled downloads has being determined that in some ocassions, if the first data block contains an ASCII 0 (zero), a call to GetPartialDownload can produce erroneous results.

To solve this problem an alternative solution have been implemented, involving the usage of this method to determine if the first data block contains the value ASCII 0 (zero) and adding them to the client application because the GetPartialDownload method deletes the problematic values at the beginning of the string automatically. For a better understanting of this solution it is suggested that you check the provided samples.

Disk downloads doesn't suffer from this problem.
GetPartialDownload

Parameters: None

Returns: String

If the value of TransferType property is TYPE_MEMORY, this method returns the content trasnferred until now.

If the value of TransferType property is TYPE_FILE, this method returns an empty string.
ClearDownLoadData

Parameters: None

Return: None

If the value of TransferType property is TYPE_MEMORY, this method clears the download buffer. This is useful to limit memory usage and to avoid getting the same data in subsequent calls to GetPartialDownload.
EndTransfer

Parameters: None

Returns: None

Ends a transfer initiated by StartHTTPTransfer.
