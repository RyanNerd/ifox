iFox.Download
Reference

Properties
Name Description
Type

Value type: Integer
Default value: DOWNLOAD_STRING

Indicates the way in which the byte sequence would be returned. Allowed values are:

DOWNLOAD_STRING: The Data property should be set with the byte sequence to return.

DOWNLOAD_FILE: The Data property should be set with the filename containing the byte sequence to return.
FileName

Value type: String

Indicates the name to assign to the generated "virtual" file. When using iFox.Download to generate a downloadable file, this is the name that would be shown to the user. If the Type property has the DOWNLOAD_FILE value and this property is not set, the physical filename will be used.
Data

Value type: String

Contains the byte sequence to return. If the Type property has the DOWNLOAD_FILE value, this property should be set with the filename containing the byte sequence to return.


Methods
Name Description
Send

Parameters: None

Returns: None

Sends the byte sequence indicated in the Data property simulating a physical file. According to the way iFox.Download is used, the file can be hosted inside a web page or downloaded for the user.

Note: An ASP page using iFox.Download should not generate another kind of response trough the Response.Write method or embedded HTML code on the page, as in this case this response would be added to the generated file. HTTP headers are also altered, so in case you sent any response before calling this method, a runtime error will arise.
