iFox.Connect
Reference

Properties
Name Description
Connections Value type: Collection

After a succesful call to the EnumConnections method, this property will contain a collection of Connection objects representing the active connections.
ForceConnection Value type: Logical

If this property's value is true, then the Dial method would establish the connection automatically without user confirmation.


Methods
Name Description
Dial Parameters: cConnection as String (optional)

Returns: Long

Establishes an Internet connection. You can ask for an especific connection using the cConnection parameter. If cConnection is not specified, or it does not correspond to a valid connection, the default connection will be used.

If no connection can be made, it returns zero; otherwise it returns a connection handle that you can use to terminate the connection with the HangUp method. If the default connection is used the return value is always -1.
EnumConnections

Parameters: None

Returns: Integer

It returns the number of active connections.

If no connection exists, the method returns zero.

If the return value is equal or greater than one, the Connections property will contain a collection of Connection objetcs representing the active connections.
EnumEntries

Parameters: None

Returns: String

It returns all the available connections (those defined in Dial-up Networking).

If no available connections exists, the method returns an empty string.

If there is more than a single available connection the string would contain their names separated by Chr(13) + Chr(10) characters.
HangUp Parameters: nConnection as Long (optional)

Returns: Logical

Closes the active Internet connection. nConnection indicates the handle corresponding to the connection to terminate. If not specified the default connection would be closed.
IsConnected Parameters: None

Returns: Logical

Determines if an active Internet connection exists.
Statistics Parameters: nConnection as Long, nType as Integer

Returns: Long

Allows to get statistics over an active connection.

The nConnection parameter must be the connection handle, obtained trough the Dial o EnumConnections methods. On Windows 95/98/Me this parameter is ignored as those operating systems doesn't carry statistical information by connection.

The nType parameter indicates the statistical type to get. Values available for this parameter are:

1. Connection speed (in bps)
2. Bytes received
3. Bytes sent
4. Connection time (in milliseconds). Under Windows 95/98/Me the returned value in this case is zero due to operating system limitations.
