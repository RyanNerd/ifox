iFox.Resume&Go
Referencia

Propiedades
Nombre Descripción
TransferType Tipo de valor: Integer
Valor por Defecto: TYPE_FILE

Si el valor de esta propiedad es TYPE_FILE la descarga se efectúa directamente al archivo indicado en la propiedad FileName.

Si el valor de esta propiedad es TYPE_MEMORY la descarga se efectúa en memoria y puede ser recuperada mediante el método GetPartialDownload.
FileName Tipo de valor: String

Si el valor de la propiedad TransferType es TYPE_FILE en esta propiedad se debe indicar el nombre del archivo local en donde se descargará el archivo remoto.

Si el valor de la propiedad TransferType es TYPE_MEMORY esta propiedad es ignorada.
FileSize Tipo de valor: Long

Luego de una llamada exitosa al método StartHTTPTransfer esta propiedad contendrá el tamaño total en bytes del archivo a descargar.
DownloadSize Tipo de valor: Long

Luego de una llamada exitosa al método StartHTTPTransfer esta propiedad contendrá la cantidad de bytes a descargar.

En caso de una transferencia parcial la propiedad FileSize indicará el tamaño total del archivo y la propiedad DownloadSize la cantidad de bytes restantes a descargar.
DownloadComplete Tipo de valor: Logical

Indica si la transferencia se ha completado.
DownloadedBytes Tipo de valor: Long

Indica la cantidad de bytes transferidos hasta el momento. El valor de esta propiedad variará entre 0 y el valor indicado por la propiedad DownloadSize.
PreviousDownload Tipo de valor: Long

Indica la cantidad de bytes transferidos en una descarga anterior. La suma de esta propiedad con el valor indicado en la propiedad DownloadedBytes indicará el tamaño total descargado en todas las sesiones.
ErrorNumber Tipo de valor: Long

En caso de producirse un error esta propiedad contendrá el número de error ocurrido. Los valores posibles son:

1. Imposible inicializar iFox.Sockets.
2. Imposible obtener la dirección IP del servidor indicado.
3. Imposible establecer comunicación con el servidor indicado.
4. No se ha obtenido respuesta del servidor.
5. El servidor no soporta el protocolo HTTP 1.1
6. El servidor no soporta descargas parciales.
7. El archivo ya ha sido completamente descargado.
8. No se ha indicado el archivo local indicado en la propiedad FileName.
9. No se ha podido abrir el archivo indicado en la propiedad FileName.
10. La respuesta obtenida del servidor es inválida.

101. Error de comunicación (sólo en DownloadNextPart).
102. Se ha producido un error de escritura (sólo en DownloadNextPart).


Métodos
Nombre Descripción
StartHTTPTransfer Parámetros: cServer as String, cURL as String, nStart as Long (opcional), nPort as Integer (opcional), nTimeOut as Integer (opcional), lDynamic as Boolean (opcional)

Respuesta: Logical

Devuelve el contenido de la URL indicada por el parámetro cURL. El nombre del servidor debe ser indicado en el parámetro cServer.

El parámetro nStart indica la posición donde debe comenzar la descarga, si no es indicado el archivo es completamente descargado.

El parámetro nPort indica el número de puerto a utilizar, en caso de no ser especificado se utilizará el puerto estandar establecido en el protocolo HTTP: 80.

El parámetro nTimeOut indica el tiempo en segundos para esperar la respuesta del servidor, si no es indicado el componente esperará por 15 segundos.

En algunas ocasiones iFox.Resume&Go no es capaz de descargar correctamente archivos generados dinámicamente (por ASP, PHP, etc.). Esto es ocasionado porque los encabezados HTTP enviados son diferentes. En estos casos puede utilizar el parámetro lDynamic para indicar que el archivo ha sido generado dinámicamente.
DownloadNextPart Parámetros: Ninguno

Respuesta: String

Continúa una transferencia iniciada con StartHTTPTransfer.

La respuesta obtenida indicará si se ha podido efectuar la operación exitosamente.

GetZeroCount Parámetros: Ninguno

Respuesta: Integer

En pruebas efectuadas en descargas realizadas a memoria se ha determinado que en ocasiones si el primer byte del bloque de datos contiene el valor ASCII 0 (cero) se producen resultados erróneos al efectuar una llamada al método GetPartialDownload.

Para solucionar este inconveniente se ha implementado una solución alternativa que involucra utilizar este método para determinar si el primer byte del bloque de datos contiene el valor ASCII 0 (cero) y agregarlos en la aplicación cliente ya que el método GetPartialDownload elimina los valores problemáticos al inicio de la cadena automáticamente. Para una mejor comprensión de la solución se sugiere consultar los ejemplos brindados.

Las descargas a disco no sufren de este inconveniente.
GetPartialDownload Parámetros: Ninguno

Respuesta: String

Si el valor de la propiedad TransferType es TYPE_MEMORY devuelve el contenido transferido hasta el momento.

Si el valor de la propiedad TransferType es TYPE_FILE en devuelve una cadena vacía.
ClearDownLoadData Parámetros: Ninguno

Respuesta: Ninguna

Si el valor de la propiedad TransferType es TYPE_MEMORY limpia el buffer de descarga. Esto resulta útil para limitar el uso de memoria y para no volver a obtener los mismos datos en sucesivas llamadas al método GetPartialDownload.
EndTransfer Parámetros: Ninguno

Respuesta: Ninguna

Finaliza una transferencia iniciada por StartHTTPTransfer.
