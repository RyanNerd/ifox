Propiedades
Nombre Descripción
SenderName Tipo de valor: String

Nombre del remitente del E-Mail. Este es el nombre que verá el receptor del mensaje.
SenderEMail Tipo de valor: String

E-Mail del remitente. Esta es la dirección que verá el receptor del mensaje.
Subject Tipo de valor: String

Asunto del mensaje.
Priority Tipo de valor: Integer
Valor por Defecto: SMTP_PRIORITY_NORMAL

Prioridad del mensaje.
NotificationEMail Tipo de valor: String

Permite solicitar al receptor del mensaje una confirmación de recepción. En la propiedad se debe colocar la dirección de E-Mail a la cual se le debe enviar la confirmación.
IncludeImages Tipo de valor: Logical

Indica que las imágenes deben ser incluidas en el mensaje. Esta propiedad solamente se aplica a los mensajes HTML, para los mensajes en modo texto será ignorada.

Si esta propiedad está en Verdadero antes de enviar el mensaje se buscará todas las ocurrencias del tag <img> y las imágenes referenciadas se agregarán al mensaje. Se debe incluir la ruta completa donde se encuentra el archivo. En caso de que la propiedad HTMLBodyType contenga el valor SMTP_BODY_FILE las imágenes pueden estar en la misma carpeta que el cuerpo. No se soportan las rutas relativas.

Se debe tener cuidado al utilizar esta opción, ya que las imágenes son incluidas junto con el mensaje y su tamaño puede crecer más de lo recomendable.
IncludeMusic Tipo de valor: Logical

Indica que música de fondo debe ser incluida en el mensaje. Esta propiedad solamente se aplica a los mensajes HTML, para los mensajes en modo texto será ignorada.

Si esta propiedad está en Verdadero antes de enviar el mensaje se buscará todas las ocurrencias del tag <bgsound> y los archivos de sonido referenciados se agregarán al mensaje. Se debe incluir la ruta completa donde se encuentra el archivo. En caso de que la propiedad HTMLBodyType contenga el valor SMTP_BODY_FILE las imágenes pueden estar en la misma carpeta que el cuerpo. No se soportan las rutas relativas.

Se debe tener cuidado al utilizar esta opción, ya que los archivos de sonido son incluidos junto con el mensaje y su tamaño puede crecer más de lo recomendable.
Body Tipo de valor: String

Contiene el cuerpo del mensaje en modo texto. Si la propiedad BodyType contiene el valor SMTP_BODY_FILE esta propiedad debe establecerse con el nombre de un archivo que contenga el texto del mensaje.
BodyCharset Tipo de valor: String
Valor por Defecto: iso-8859-1

Indica la codificación aplicada al cuerpo del mensaje. Este valor no se utiliza para ningún tipo de conversión y se supone que el mensaje ya se proporciona codificado apropiadamente. No obstante iFox.SMTP se encarga de codificar todos los mensajes para utilizar un máximo de 7 bits de forma que puedan ser procesados por cualquier servidor SMTP o cliente de correo.
BodyType Tipo de valor: String
Valor por Defecto: SMTP_BODY_STRING

Indica la forma en que se proporciona el cuerpo del mensaje. Los valores permitidos son:

SMTP_BODY_STRING: Se debe establecer la propiedad Body con el texto del mensaje.

SMTP_BODY_FILE: Se debe establecer la propiedad Body con el nombre de un archivo que contenga el mensaje.
HTMLBody Tipo de valor: String

Contiene el cuerpo del mensaje en formato HTML. Si la propiedad HTMLBodyType contiene el valor SMTP_BODY_FILE esta propiedad debe establecerse con el nombre de un archivo que contenga el texto del mensaje.

No todos los clientes de correo soportan mensajes en este formato, así que resulta recomendable establecer la propiedad Body con una versión en texto puro del mensaje. iFox.SMTP es capaz de codificar el mensaje incluyendo los dos formatos.

Si esta propiedad contiene una cadena vacía se enviará el mensaje en modo texto.
HTMLBodyCharset Tipo de valor: String
Valor por Defecto: iso-8859-1

Similar a la propiedad BodyCharset pero aplicable a los mensajes en formato HTML.
HTMLBodyType Tipo de valor: String
Valor por Defecto: SMTP_BODY_STRING

Similar a la propiedad BodyType pero aplicable a los mensajes en formato HTML.
RTFBody Tipo de valor: String

Contiene el cuerpo del mensaje en formato RTF. Si la propiedad RTFBodyType contiene el valor SMTP_BODY_FILE esta propiedad debe establecerse con el nombre de un archivo que contenga el texto del mensaje.

No todos los clientes de correo soportan mensajes en este formato, así que resulta recomendable establecer la propiedad Body con una versión en texto puro del mensaje. iFox.SMTP es capaz de codificar el mensaje incluyendo los dos formatos.

Si esta propiedad contiene una cadena vacía se enviará el mensaje en modo texto.
RTFBodyCharset Tipo de valor: String
Valor por Defecto: iso-8859-1

Similar a la propiedad BodyCharset pero aplicable a los mensajes en formato RTF.
RTFBodyType Tipo de valor: String
Valor por Defecto: SMTP_BODY_STRING

Similar a la propiedad BodyType pero aplicable a los mensajes en formato RTF.
ErrorNumber Tipo de valor: Long

En caso de producirse un error esta propiedad contendrá el número de error ocurrido. Los valores posibles son:

1. Error de conexión
2. No se ha obtenido respuesta del servidor SMTP
3. La dirección de E-Mail del remitente es inválida
4. No se ha indicado ningún destinatario
5. El servidor SMTP ha respondido con un mensaje de error.
6. El servidor SMTP ha rechazado la dirección de E-Mail del remitente.
7. El servidor SMTP ha rechazado alguno de los destinatarios.
8. El servidor SMTP utiliza un método de autenticación no soportado por iFox.
9. Error durante la autenticación. El servidor SMTP ha rechazado el nombre de usuario.
10. Error durante la autenticación. El servidor SMTP ha rechazado la clave de acceso.
11. Error durante la autenticación. El servidor SMTP ha rechazado el nombre de usuario o la clave de acceso.
12. Error al intentar solicitar al servidor que cancele el mensaje actual.
999. Se ha llamado al método SendNextPart luego que todo el mensaje ha sido transmitido.
ExtraErrorInfo Tipo de valor: String

Contiene una descripción del error ocurrido. El valor contenido dependerá del código de error indicado por la propiedad ErrorNumber.

3. Dirección de E-Mail del remitente.
5. Respuesta obtenida del servidor SMTP.
6. Respuesta obtenida del servidor SMTP.
7. Dirección de E-Mail del destinatario.

Para los otros tipos de error esta propiedad contendrá una cadena vacía.
SMTPErrorNumber Tipo de valor: Long

Si la propiedad ErrorNumber contiene los valores 5, 6 ó 7 esta propiedad contendrá el código de error devuelto por el servidor SMTP.
SMTPResponses Tipo de valor: String

Esta propiedad contendrá todas las respuestas obtenidas del servidor SMTP. Esta información puede resultar útil para la detección de errores o creación de logs.
PartSize Tipo de valor: Long
Valor por Defecto: 1024

Cantidad de bytes a enviar en cada llamada al método SendNextPart.

No se debe modificar el valor de esta propiedad luego de comenzado el envío del mensaje.
EOD Tipo de valor: Logical

End of Data. Al efectuar un envío a través del método SendNextPart esta propiedad indicará si ya se ha terminado de enviar el mensaje.
ValidAddresses Tipo de valor: Logical

Por defecto iFox.SMTP efectúa una validación básica de las direcciones de E-Mail utilizadas rechazando aquellas que sean inválidas. Si no se desea efectuar esta validación establezca esta propiedad en False. La validación que se realiza controla solamenta algunas características sintácticas de la dirección y su aceptación no significa que sea realmente una dirección de E-Mail válida.
BodyData Tipo de valor: String

Luego de una llamada al método PrepareMail esta propiedad contendrá el texto del mensaje a enviar. De esta forma es posible realizar una vista previa del mensaje.
AuthenticationMethod Tipo de valor: Integer

Indica el método de autenticación a utilizar en el caso que el servidor SMTP requiera autenticación. Los valores posibles son:

0. Sin autenticación.
1. Autenticación por POP3. Aunque no muy común esta modalidad de autenticación aún es utilizada por algunos servidores. En esta modalidad el cliente establece una conexión con un servidor POP3 determinado, si esta conexión es exitosa se permite la utilización del servidor SMTP.
2. Autenticación por SMTP.
UserName Tipo de valor: String

Indica el nombre de usuario a utilizar para efectuar la autenticación ante el servidor SMTP. En el caso de establecer la propiedad AuthenticationMethod en 1 se debe asignar el nombre de usuario del servidor POP3.
Password Tipo de valor: String

Indica la clave de acceso a utilizar para efectuar la autenticación ante el servidor SMTP. En el caso de establecer la propiedad AuthenticationMethod en 1 se debe asignar la clave de acceso del servidor POP3.
POP3Server Tipo de valor: String

En el caso de establecer la propiedad AuthenticationMethod en 1 se debe establecer esta propiedad con la dirección del servidor POP3.


Métodos
Nombre Descripción
AddRecipient Parámetros: cName as String, cEMail as String

Respuesta: Logical

Agrega un destinatario al mensaje. Si se desea enviar el mensaje a múltiples destinatarios se deberá llamar a este método por cada uno de ellos.

La respuesta obtenida indicará si el remitente ha sido aceptado o rechazado por que la dirección de E-Mail es inválida.
AddCC Parámetros: cName as String, cEMail as String

Respuesta: Logical

Este método es similar a AddRecipient pero aplicable a los receptores del tipo CC.
AddBCC Parámetros: cName as String, cEMail as String

Respuesta: Logical

Este método es similar a AddRecipient pero aplicable a los receptores del tipo BCC (ocultos).

Como el nombre del receptor no es necesario en este caso el parámetro cName será ignorado. Este parámetro se reserva para futuras versiones de iFox.SMTP.
AddAttachment Parámetros: cFile as String

Respuesta: Logical

Adjunta el archivo indicado por el parámetro cFile al mensaje. Si se desea adjuntar múltiples archivos se deberá llamar a este método por cada uno de ellos.

La respuesta obtenida indicará si el archivo ha sido aceptado o rechazado por que no existe.

ValidMailAddress Parámetros: cAddress as String

Respuesta: Logical

Indica si una dirección de E-Mail es válida. Este método no puede indicar si la dirección indicada existe realmente sino que verifica que se cumplan algunas reglas básicas. Por ejemplo: que contenga una arroba.
NewMessage Parámetros: cAddress as String

Respuesta: Ninguna

Reinicializa el objeto y lo prepara para enviar otro mensaje. Todas las propiedades vuelven a sus valores por defecto.
ConnectSend Parámetros: cServer as String, nPort as Long (opcional)

Respuesta: Logical

Esta es la función de envío de más alto nivel. Se conecta al servidor SMTP, luego envía el mensaje y finalmente se desconecta.

El parámetro cServer debe contener el nombre o dirección IP de un servidor SMTP. El parámetro nPort indica el número de puerto a utilizar, en caso de no ser especificado se utilizará el puerto estandar establecido en el protocolo SMTP: 25.

La respuesta obtenida indica si el mensaje ha podido ser enviado exitosamente.
Connect Parámetros: cServer as String, nPort as Long (opcional)

Respuesta: Logical

Establece una conexión con un servidor SMTP.

El parámetro cServer debe contener el nombre o dirección IP de un servidor SMTP. El parámetro nPort indica el número de puerto a utilizar, en caso de no ser especificado se utilizará el puerto estandar establecido en el protocolo SMTP: 25.

Antes de liberar el objeto se debe cerrar la conexión utilizando el método Disconnect.

La respuesta obtenida indica si la conexión ha podido establecerse exitosamente.
Disconnect Parámetros: lFastDisconnect as Logical (opcional)

Respuesta: Logical

Cierra una conexión establecida utilizando el método Connect.

La respuesta obtenida indica si se ha producido un error al intentar cerrar la conexión.

El parámetro lFastDisconnect indica que no se debe esperar confirmación del servidor. Es posible utilizar este parámetro para mejorar la velocidad de respuesta cuando se está interrumpiendo el envío de un mensaje. Cuando lFastDisconnect tiene el valor Verdadero la respuesta del método y el contenido de la propiedad ErrorNumber pueden ser incorrectos.
Send Parámetros: Ninguno

Respuesta: Logical

Envia el mensaje. Se debe establecer previamente una conexión con un servidor SMTP mediante el método Connect.

Otro mensaje puede ser enviado utilizando la misma conexión llamando al método NewMessage. Se debe tener en cuanta que algunos servidores SMTP no envían el mensaje inmediatamente sino que lo colocan en una cola de espera hasta que se cierre la conexión. Se debe tener cuidado para no superar la cantidad máxima de mensajes en espera.

La respuesta obtenida indica si se ha producido un error al enviar el mensaje.
StartPartialSending Parámetros: Ninguno

Respuesta: Logical

Comienza un envío parcial. Este método de envío puede ser útil para enviar un mensaje de gran tamaño sin que la aplicación deje de responder al usuario o cuando se desea mostrar una barra de progreso.

La respuesta obtenida indica si se ha producido un error.
SendNextPart Parámetros: nRetries as Integer (opcional)

Respuesta: Logical

Envía la siguiente parte de un mensaje. Previamente se debe haber llamado al método StartPartialSending. Este método debe ser llamado repetidamente hasta que la propiedad EOD indique que el mensaje ha sido enviado en su totalidad. El tamaño de cada parte se encuentra indicado en la propiedad PartSize. Luego de finalizado el envío se debe llamar al método EndPartialSending.

El parámetro nRetries permite indicar la cantidad de veces que debe intentarse el envío. Ajustar este valor puede evitar errores de transferencia cuando se utiliza el componente sobre una mala conexión a internet. El valor por defecto de este parámetro es 3.

La respuesta obtenida indica si se ha producido un error.
EndPartialSending Parámetros: Ninguno

Respuesta: Logical

Finaliza el envío parcial de un mensaje.

La respuesta obtenida indica si se ha producido un error.
GetCurrentPosition Parámetros: Ninguno

Respuesta: Long

Indica la posición actual del puntero durante un envío parcial. Obviamente, este valor será menor o igual al obtenido mediante el método GetMessageSize. El valor obtenido solamente es válido luego de una llamada exitosa al método StartPartialSending.
GetMessageSize Parámetros: Ninguno

Respuesta: Long

Indica el tamaño en bytes del mensaje. El valor obtenido solamente es válido luego de una llamada exitosa a los métodos StartPartialSending, Send o ConnectSend.
PrepareMail Parámetros: Ninguno

Respuesta: Ninguna

Construye el mensaje a enviar. Luego de una llamada a este método la propiedad BodyData contendrá el código del mensaje a enviar. De esta forma es posible realizar una vista previa del mensaje.
