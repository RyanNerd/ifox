Nombre Descripción
Parser Tipo de valor: String
Valor por defecto: Msxml2.DOMDocument.4.0

Para el funcionamiento del método XMLtoCursor iFox.XML utiliza el parser XML de Microsoft (www.msdn.microsoft.com/xml). Por defecto iFox.XML utiliza la versión 4 pero es posible indicar que se debe utilizar otra versión estableciendo esta propiedad con el ID de la clase.
XSDType Tipo de valor: Numérico
Valor por defecto: XSD_INLINE

Indica como debe generarse el esquema:
XSD_NONE: No se genera ningún tipo de esquema.
XSD_INLINE: Se genera un esquema "inline" dentro del código XML generado.
XSD_EXTERNAL: No implementado. Se reserva para futuras versiones.


Métodos
Nombre Descripción
AddTable Parámetros: cTable as String

Respuesta: Ninguna

Agrega una tabla para que sea incluida en el código XML generado por el método CursortoXML.
CursortoXML Parámetros: cFile as String (opcional)

Respuesta: Lógical / String

Genéra el código XML correspondiente al contenido de las tablas indicadas utilizando el método AddTable.

Si se indica el parámetro cFile el resultado será escrito directamente en ese archivo, eliminando todo contenido existente. La respuesta será del tipo lógico e indicará si se ha podido generar el archivo exitosamente.

Si no se indica el parámetro cFile el resultado será del tipo string y contendrá el código XML generado.

XMLToCursor Parámetros: cXML as String, cCursorList as String (opcional)

Respuesta: Logical

Este método convierte el código XML indicado en el parámetro cXML en uno o más cursores.

En el parámetro cCursorList es posible indicar el nombre que tomará el cursor generado. Si el código XML incluye más de un cursor se debe indicar los nombres deseados separándolos con comas. Si este parámetro no es indicado se utilizarán los nombres asignados en el código XML por lo que solamente es recomendable ignorarlo cuando se sabe que el código ha sido generado utilizando una nomenclatura conocida. Si el código XML incluye más cursores que los indicados en cCursorList estos serán ignorados, de esta forma se evita que un archivo XML malformado genere una cantidad excesiva de cursores.

Si el código XML no posee un esquema todos los campos del cursor resultante serán del tipo caracter ya que será imposible establecer el tipo correcto.

Nota: Este método requiere de al menos la versión 3 del parser XML de Microsoft. Consultar también la documentación de la propiedad Parser.
