Propiedades
Nombre Descripción
ErrorNumber Tipo de valor: Long

En caso de producirse un error al ejecutar cualquiera de los métodos esta propiedad contendrá el número del error ocurrido.
ConnectionType Tipo de valor: Integer

Indica la manera en que se accede a Internet. Los valores posibles son:

0. Se utiliza la configuración indicada en el Panel de Control de Windows. Este es el valor por defecto.
1. Se utiliza una conexión directa a Internet.
3. Se utiliza un proxy para acceder a Internet.
ProxyServer Tipo de valor: String

La dirección (y opcionalmente el puerto) del servidor proxy a utilizar cuando ConnectionType tiene el valor 3.

Ejemplos:
192.168.0.90
192.168.0.90:750
ProxyUserName Tipo de valor: String

El nombre de usuario utilizado para identificarse ante el servidor proxy cuando ConnectionType tiene el valor 3. Si el servidor proxy no requiere autenticación dejar esta propiedad en blanco.
ProxyPassword Tipo de valor: String

La clave utilizada para identificarse ante el servidor proxy cuando ConnectionType tiene el valor 3. Si el servidor proxy no requiere autenticación dejar esta propiedad en blanco.
IgnoreSSLErrors Tipo de valor: Lógico

Indica que se permite acceder a páginas que tengan problemas con su certificado de seguridad. Permitir este acceso no necesariamente implica un riesgo, lo cual dependerá del problema que tenga el certificado (el cual puede determinar navegando el sitio) y la confiabilidad del sitio a acceder.

El valor por defecto de esta propiedad es Falso.
TransferSize Tipo de valor: Long

Luego de una llamada existosa a StartAsyncGet o StartAsyncPost contiene la cantidad de bytes a transferir. Esta información puede ser utilizada para actualizar una barra de progreso.
CompletedTransfer Tipo de valor: Long

Luego de una llamada existosa a StartAsyncGet o StartAsyncPost contiene la cantidad de bytes ya transferidos. Esta información puede ser utilizada para actualizar una barra de progreso.
EOT Tipo de valor: Logical

Luego de una llamada existosa a StartAsyncGet o StartAsyncPost indica si la transferencia ha sido completada.


Métodos
Nombre Descripción
Get Parámetros: cURL as String

Respuesta: String

Devuelve el contenido de la URL indicada por el parámetro cURL
StartAsyncGet Parámetros: cURL as String

Respuesta: Logical

Devuelve el contenido de la URL indicada por el parámetro cURL en forma asincrónica. Este método permite monitorear el progreso de la transferencia.
ContinueAsyncGet Parámetros: Ninguno

Respuesta: Logical

Continúa una transferencia iniciada con StartAsyncGet. Ejecute este método repetidamente hasta que la propiedad EOT contenga el valor Verdadero.

La respuesta obtenida indicará si se ha podido efectuar la operación exitosamente.
EndAsyncGet Parámetros: Ninguno

Respuesta: Ninguna

Finaliza una transferencia iniciada con StartAsyncGet.
ClearPostFields Parámetros: Ninguno

Respuesta: Ninguna

Reinicializa el objeto y lo prepara para simular el envío de otro formulario.
AddPostField Parámetros: cField as String, cValue as String

Respuesta: Ninguna

Agrega un campo a un formulario "virtual" que luego será utilizado para simular una entrada de datos en una página web.

El parámetro cField indica el nombre del campo y el parámetro cValue indica su contenido.
AddPostFile Parámetros: cField as String, cFile as String

Respuesta: Ninguna

Simular al método AddPostField pero simula un campo del tipo File utilizado para subir un archivo.
AddPostFieldasFile Parámetros: cField as String, cValue as String, cFile as String

Respuesta: Ninguna

Similar al método AddPostFile pero en lugar de utilizar un archivo real existente en un disco local lo simula. El parámetro cValue se debe establecer con el contenido del archivo mientras que el parámetro cFile indica el nombre de archivo a utilizar.
Post Parámetros: cServer as String, cURL as String, cUsername as String (opcional), cPassword as String (opcional), nPort as Long (opcional)

Respuesta: String

Simula el envío de un formulario utilizando los campos indicados por los métodos AddPostField y AddPostFile.

Los parámetros cServer y cURL indican el nombre del servidor y la URL (excluyendo el nombre del servidor obviamente) donde se encuentra la página que recibirá los datos enviados.

En caso de ser necesario es posible indicar un nombre de usuario y clave de acceso a utilizar mediante los parámetros cUserName y cPassword.

El parámetro nPort indica el número de puerto a utilizar, en caso de no ser especificado se utilizará el puerto estandar establecido en el protocolo HTTP: 80.
StartAsyncPost Parámetros: cServer as String, cURL as String, cUsername as String (opcional), cPassword as String (opcional), nPort as Long (opcional)

Respuesta: Logical

Simula el envío de un formulario utilizando los campos indicados por los métodos AddPostField y AddPostFile en forma asincrónica. Este método permite monitorear el progreso de la transferencia.
ContinueAsyncPost Parámetros: Ninguno

Respuesta: Logical

Continúa una transferencia iniciada con StartAsyncPost. Ejecute este método repetidamente hasta que la propiedad EOT contenga el valor Verdadero.

La respuesta obtenida indicará si se ha podido efectuar la operación exitosamente.
EndAsyncPost Parámetros: Ninguno

Respuesta: Ninguna

Finaliza una transferencia iniciada con StartAsyncPost. La respuesta del servidor se obtiene en este momento, por lo que la ejecución de este método puede demorar dependiendo del tamaño de la respuesta y la velocidad de conexión.
URLEncode Parámetros: cURL as String

Respuesta: String

Codifica la cadena enviada en el parámetro cURL para que sea una URL válida, por ejemplo reemplazando los espacios por %20.
